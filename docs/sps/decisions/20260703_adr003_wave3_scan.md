## 1. 需求深挖与质疑 (Adversarial Questioning)

### 1.1 为什么要进行 Wave 3 扩展？
前两波已经完成了底层原理解析和大厂面试题的从零到一建立。然而，对于“初二编程小白为了当大厂 CTO”这一极端目标，缺乏直观、纳米级的**物理结构图与执行流程图**（有图、有逻辑）。
纯文字即使再白话，对小白而言依然存在认知死角。我们需要用极具黑客硬核美感的 **ASCII 艺术流程图** 在每个核心知识点旁进行可视化装配。

### 1.2 注入方式是否会破坏现有功能？
我们必须维持 **Deletions 始终为 0** 的铁律。
- 所有正文章节依然采用定位到末尾的 `> **下一章**`，并在其上方进行**增量合并式追加**。
- 采用新的 `wave3_data` 结构进行 runner 驱动，做精准的防重入判定，绝不破坏 Wave 1 和 Wave 2 已经通过测试的既有内容。

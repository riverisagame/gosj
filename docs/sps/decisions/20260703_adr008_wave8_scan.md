# ADR 008: Wave 8 Markdown 静态语法健全性与闭合性安全审计决策 (SCAN 阶段)

- **日期**：2026-07-03
- **决策状态**：提案中
- **触达范围**：`ch01` ~ `ch13` 及附录 `appendix-go-versions.md` 全量 Markdown 文件。

## 1. 需求深挖与质疑 (Adversarial Questioning)

### 1.1 为什么需要第八波（Wave 8）Markdown 静态语法健全性审计？
经过 Wave 1 至 Wave 7 的多轮定位增量注入，原有的 14 个 Markdown 文件中插入了大量的：
1. ASCII 流程图（包含特殊制表符）；
2. LaTeX 数学公式（带有 `$` 和 `$$`）；
3. Cgo / 汇编 / Go 的多代码块（```` ` ```` 围栏）；
4. HTML 注释锚点（`<!-- @Ref: ... -->`）。

如果在多次重入拼接过程中，某些行由于定位偏移导致了代码块 ``` 未正常闭合，或者 LaTeX 符号配对失效，这会在 Markdown 渲染引擎（如 GitHub 或 IDE 预览）中引发排版大面积塌陷，严重影响读者的阅读体验。
作为一个严谨的编译器，必须在发布前建立 Markdown 词法分析与边界健全性（Boundary Check）的静态对冲工具。

### 1.2 审计机制设计
我们将在 `test_tutorial_extensions.py` 中集成 Markdown 的词法状态机检查：
1. **代码块闭合校验**：统计每个文件中 ``` 字符的出现次数，必须是偶数（即每次开启都必须有闭合配对）；
2. **LaTeX 公式对称校验**：统计行内单个 `$` 以及块级 `$$` 的配对性；
3. **HTML 注释完备性**：校验所有的注释标志是否成对存在 `<!--` 与 `-->`。

---

## 2. 拟升级的静态审计指标大纲

| 审计项 | 物理检验逻辑 | 异常对冲方案 |
|---|---|---|
| **代码围栏闭合** | 检测全局 `` ``` `` 出现频次是否为 $2N$ | 抛出语法中断，精确定位未闭合文件与行号 |
| **HTML 标签闭合** | 校验注释标签 `<!--` 与 `-->` 的数量是否绝对相等 | 报警并阻断 CI，定位拼写错误的注释锚点 |
| **数学公式配对** | 校验 LaTeX `$` 字符在单行或跨行中是否对称闭合 | 捕获不对称行，防止公式在渲染时污染正文文本 |

## 3. 回归与验证对齐计划

我们将在此 ADR 获得确认后，升级 `test_tutorial_extensions.py`：
1. 集成 `check_markdown_syntax()` 词法校验引擎。
2. 确保在执行 `python test_tutorial_extensions.py` 时，该静态审计引擎自动为 14 个文件做全量格式扫描。
3. 确保 100% 格式无误后，再将项目交付。
